# Cloud Computing

## Definition

Cloud Computing is the delivery of computing services such as servers, storage, databases, networking, and software over the internet instead of using local computers or physical servers.

It allows users to access resources whenever needed and pay only for what they use.

## Why is it used?

- Reduces infrastructure cost.
- Access resources from anywhere.
- Scale resources easily.
- Faster application deployment.
- Improves business flexibility.

## Key Features

- On-demand access – Resources are available whenever needed.
- Pay-as-you-go – Pay only for the resources used.
- Scalability – Increase or decrease resources easily.
- High Availability – Services are available most of the time.
- Global Access – Access services from anywhere.

## Advantages

- Lower infrastructure cost.
- Easy scalability.
- Faster deployment.
- Automatic updates.
- Better disaster recovery.

## Disadvantages

- Internet dependency.
- Security concerns.
- Vendor lock-in.
- Downtime may affect services.

## Components / Types / Workflow

- Compute
- Storage
- Networking
- Database
- Security
- Monitoring

## Real-world Example

A company hosts its website on Microsoft Azure instead of purchasing physical servers.

## Interview Tip

"Cloud Computing means using IT resources over the internet instead of managing physical infrastructure."

## Quick Revision

- Internet-based computing.
- Pay-as-you-go pricing.
- Highly scalable.
- Accessible from anywhere.
- Reduces infrastructure management.
- Supports high availability.

# Cloud Service Models

## Definition

Cloud Service Models define the level of service provided by the cloud provider.

## Why is it used?

- Different business needs.
- Reduce management effort.
- Faster application deployment.
- Flexible service options.

## Key Features

| Model | User Manages | Provider Manages |
|--------|--------------|------------------|
| IaaS | OS, Apps | Hardware |
| PaaS | Applications | Platform |
| SaaS | Only Uses Software | Everything |
| FaaS | Function Code | Infrastructure & Runtime |

## Advantages

- Flexible options.
- Lower maintenance.
- Faster development.

## Disadvantages

- Less control in higher-level services.
- Vendor dependency.

## Components / Types / Workflow

- IaaS
- PaaS
- SaaS
- FaaS

## Real-world Example

Azure offers Virtual Machines, App Service, Microsoft 365, and Azure Functions.

## Interview Tip

"As we move from IaaS to SaaS, the cloud provider manages more components."

## Quick Revision

- Four service models.
- Different management levels.
- Flexible deployment.
- Cost-efficient.

## Important Interview Questions

- What are Cloud Service Models?
- Compare IaaS, PaaS, SaaS.
- Which model provides maximum control?
- Which model requires least management?

# FaaS (Function as a Service)

## Definition

FaaS is a serverless service where developers write small functions, and the cloud provider executes them only when triggered.

## Why is it used?

- Event-driven applications.
- Pay only when code runs.
- No server management.
- Automatic scaling.

## Key Features

- Serverless.
- Event triggered.
- Auto scaling.
- Pay per execution.

## Advantages

- Low cost.
- Easy deployment.
- Highly scalable.
- No infrastructure management.

## Disadvantages

- Cold start delay.
- Execution time limits.
- Vendor lock-in.

## Components / Types / Workflow

```text
Event
   ↓
Function Trigger
   ↓
Execute Code
   ↓
Return Response
```

## Real-world Example

Azure Functions automatically process uploaded files in Azure Storage.

## Interview Tip

"FaaS runs individual functions on demand without managing servers."

## Quick Revision

- Serverless computing.
- Event driven.
- Pay per execution.
- Automatic scaling.
- No server management.

- What is FaaS?
- How is FaaS different from PaaS?

- Give an example of FaaS.
